## 1. 项目概述

### 1.1 项目简介
Contoso.tech 企业在线培训平台是一个面向企业的现代化在线学习管理系统（LMS），旨在帮助企业构建高效的内部培训体系。该平台采用前后端分离的架构设计，提供完整的课程管理、学习跟踪和用户管理功能。

### 1.2 核心价值
- **企业培训数字化**: 将传统培训转型为在线学习模式
- **学习效果可量化**: 实时跟踪学习进度和完成情况
- **管理效率提升**: 统一的课程和用户管理平台
- **灵活扩展性**: 支持未来功能扩展和多端适配

### 1.3 主要功能模块
- 用户管理系统（注册、登录、权限控制）
- 课程管理系统（创建、编辑、发布、分类）
- 学习功能（浏览、学习、进度跟踪、认证）
- 系统管理（用户管理、数据统计、系统配置）

## 3. 数据库架构设计

### 3.1 数据库选型
采用 **MySQL 8.0** 作为主数据库，原因如下：
- 关系型数据适合结构化的课程和用户数据
- 事务支持确保数据一致性
- 性能稳定，可满足中小规模应用需求
- 生态成熟，运维工具丰富

### 3.2 核心数据模型

#### 3.2.1 实体关系概览
```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│    Users    │───────│  User_Roles │───────│    Roles    │
│             │   N:M │             │   N:M │             │
└─────────────┘       └─────────────┘       └─────────────┘
       │ 1
       │
       │ N
┌─────────────┐       ┌─────────────┐
│ Enrollments │───────│   Courses   │
│             │   N:1 │             │
└─────────────┘       └─────────────┘
       │ 1                    │ N:1
       │                      │
       │ N              ┌─────────────┐
┌─────────────┐        │ Categories  │
│   Progress  │        │             │
│             │        └─────────────┘
└─────────────┘               │ 自关联
                              │ (树形结构)
```

#### 3.2.2 核心表结构

**users (用户表)**
- 存储用户基本信息（用户名、邮箱、密码哈希等）
- 支持用户激活状态、邮箱验证
- 记录最后登录时间
- 包含部门、职位等扩展字段

**roles (角色表)**
- 定义系统角色（学员、讲师、管理员）
- 存储角色权限（JSON 格式）
- 支持灵活的权限配置

**user_roles (用户角色关联表)**
- 多对多关系，支持一个用户多个角色
- 记录角色分配时间和分配人

**courses (课程表)**
- 课程基本信息（标题、描述、封面等）
- 课程状态（草稿、已发布、已归档）
- 关联讲师和分类
- 包含难度级别、时长等元数据

**categories (课程分类表)**
- 支持多级分类（通过 parent_id 自关联）
- 包含排序、图标、颜色等展示属性

**enrollments (课程注册表)**
- 学员与课程的关系
- 记录注册时间、完成状态、进度
- 支持评分和反馈

**learning_progress (学习进度表)**
- 详细的学习进度跟踪
- 记录每个课程单元的完成情况
- 支持学习时长统计

### 3.3 数据库设计原则
- **规范化**: 遵循第三范式，减少数据冗余
- **索引优化**: 在高频查询字段建立索引
- **外键约束**: 确保数据一致性和引用完整性
- **字符集**: 使用 utf8mb4 支持完整的 Unicode 字符
- **时间戳**: 记录创建和更新时间，便于审计

### 4.1 用户管理模块

#### 4.1.1 功能概述

- 邮箱验证和密码重置
- 用户资料管理
- 角色和权限管理

#### 4.1.2 认证流程
```
用户登录
    ↓
前端表单验证
    ↓
提交到后端 /api/auth/login
    ↓
验证用户名和密码
    ↓
生成 JWT Token (包含用户ID、角色等)
    ↓
返回 Token 和用户信息
    ↓
前端存储 Token (localStorage/sessionStorage)
    ↓
后续请求携带 Token (Authorization: Bearer <token>)
    ↓
后端验证 Token 有效性
    ↓
允许访问受保护资源
```

#### 4.1.3 权限控制

## 5. 安全架构

### 5.1 认证与授权

#### 5.1.1 JWT 认证机制
```json
{
  "sub": "user123",
  "name": "Zhang San",
  "email": "zhangsan@contoso.com",
  "roles": ["STUDENT", "INSTRUCTOR"],
  "iat": 1706582400,
  "exp": 1706668800
}
```

**Token 生命周期管理**：
- Access Token: 短期有效（默认 30 分钟）
- Refresh Token: 长期有效（默认 7 天）
- Token 刷新机制避免频繁登录

#### 5.1.2 Spring Security 配置
- **密码加密**: BCrypt 加密算法
- **CSRF 保护**: 对于状态改变的请求启用
- **CORS 配置**: 控制跨域资源访问
- **方法级权限**: @PreAuthorize 注解

### 5.2 数据安全

#### 5.2.1 敏感数据保护
- 密码: BCrypt 哈希（不可逆）
- 个人信息: 数据库字段级加密（计划）
- 文件上传: 类型和大小限制

#### 5.2.2 SQL 注入防护
- 使用参数化查询（JPA/Prepared Statement）
- 输入验证和清理
- 最小权限原则（数据库用户权限）

### 5.3 API 安全

#### 5.3.1 请求验证
- Token 验证（每个请求）
- 请求速率限制（防止 DDoS）
- 参数校验（JSR-380 Bean Validation）

#### 5.3.2 响应安全
- 敏感信息过滤
- 统一错误处理（不泄露系统信息）
- 安全响应头（X-Frame-Options, X-Content-Type-Options 等）

## 7. 性能优化

### 7.1 前端性能优化

#### 7.1.1 构建优化
- **代码分割**: 路由级别的懒加载
- **Tree Shaking**: 移除未使用的代码
- **资源压缩**: Gzip/Brotli 压缩
- **CDN 加速**: 静态资源 CDN 分发（计划）

#### 7.1.2 运行时优化
- **虚拟滚动**: 长列表性能优化
- **图片懒加载**: 按需加载图片
- **缓存策略**: 浏览器缓存 + Service Worker（计划）
- **骨架屏**: 改善首屏加载体验

### 7.2 后端性能优化

#### 7.2.1 数据库优化
- **索引优化**: 高频查询字段建立索引
- **查询优化**: 避免 N+1 查询，使用 JOIN
- **连接池**: HikariCP 连接池配置
- **缓存层**: Redis 缓存热点数据（Phase 2）

#### 7.2.2 应用优化
- **异步处理**: 使用 @Async 处理耗时操作
- **批量操作**: 批量插入/更新减少 DB 交互
- **分页查询**: 大数据量分页返回
- **API 响应优化**: 只返回必要字段

### 7.3 监控和日志

#### 7.3.1 应用监控（计划）
- **健康检查**: Spring Boot Actuator
- **性能指标**: Prometheus + Grafana
- **链路追踪**: Azure Application Insights
- **错误追踪**: Sentry / ELK Stack

#### 7.3.2 日志管理
- **日志级别**: DEBUG, INFO, WARN, ERROR
- **结构化日志**: JSON 格式，便于分析
- **日志聚合**: 集中式日志收集（计划）
- **日志轮转**: 按大小/时间自动轮转

- **命名规范**: 
  - 组件文件: PascalCase (e.g., `UserProfile.vue`)
  - 普通文件: camelCase
  - 常量: UPPER_SNAKE_CASE
- **代码风格**: ESLint + Prettier
- **提交规范**: Conventional Commits

#### 8.1.2 后端代码规范
- **命名规范**: 
  - 类名: PascalCase
  - 方法/变量: camelCase
  - 常量: UPPER_SNAKE_CASE
  - 包名: lowercase
- **代码风格**: Google Java Style Guide
- **注释规范**: JavaDoc

### 8.2 Git 工作流

#### 8.2.1 分支策略
```
main (生产环境)
  ├── develop (开发主分支)
  │   ├── feature/user-management (功能分支)
  │   ├── feature/course-catalog (功能分支)
  │   └── bugfix/login-issue (Bug 修复分支)
  └── hotfix/critical-security-fix (紧急修复分支)
```

#### 8.2.2 提交信息规范
```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）:
- `feat`: 新功能

- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具配置
